<figure class="image"><img height="2400" src="https://postmark-app.transforms.svdcdn.com/production/images/Postmark-Skills-Social.png?w=2400&amp;h=2400&amp;q=85&amp;auto=format&amp;fit=crop&amp;dm=1777233274&amp;s=6bbacaf8c3c09106d5fa328d79aa5f85" width="2400" /></figure><p>If you've ever asked an AI coding agent to help you integrate email into your app, you've probably gotten code that looks right but isn't quite right. Maybe it used an outdated endpoint. Maybe it forgot about Message Streams entirely. Maybe it hallucinated a method that doesn't exist. (We've seen all three.)</p><p>AI agents are only as good as the context they have, and until now, most of them had to rely on whatever fragments of Postmark documentation made it into their training data.</p><p>Today, we're releasing <a href="https://github.com/ActiveCampaign/postmark-skills">Postmark Skills</a>, a set of open source Agent Skills that teach AI coding agents how to work with Postmark correctly.</p><h2><strong>What are Agent Skills?</strong></h2><p>Agent Skills are an <a href="https://github.com/agentskills/agentskills">open standard</a> for giving AI coding agents specialized knowledge. Think of them as instruction manuals that your agent reads before writing code, except these manuals are structured specifically for how AI models process information.</p><p>Each skill is a folder with a&nbsp;SKILL.md file that contains best practices, code examples, API details, and common mistakes to avoid. When you install a skill, your AI agent loads it as context and uses it to write better, more accurate code.</p><p>Skills work with Claude Code, Cursor, GitHub Copilot, OpenAI Codex, Gemini CLI, and a growing list of other tools. If your agent can read a markdown file, it can use a skill.</p><h2><strong>What's included</strong></h2><p>Postmark Skills covers five areas of the Postmark API:</p><p><strong>Sending email</strong> — Everything your agent needs to send transactional or broadcast emails through Postmark. Single sends, batch sends (up to 500 per request), template-based sends, and the Bulk API for campaigns. The skill includes the decision logic for when to use each approach, proper Message Stream handling, error codes and retry strategies, and attachment support, including batch attachments, which some alternatives don't support.</p><p><strong>Inbound email processing</strong> — Building workflows that receive and parse incoming email. Reply-by-email threading, email-to-ticket systems, document extraction from attachments, and routing with MailboxHash. Your agent will know to use&nbsp;StrippedTextReply instead of&nbsp;TextBody for reply parsing, set the body parser limit to 50mb for messages with attachments, and always return HTTP 200 before processing.</p><p><strong>Template management</strong> — Creating and managing server-side email templates with Handlebars syntax. Layout inheritance, template validation, cross-server pushing from staging to production, and the difference between template aliases and IDs (use aliases, they survive re-creation and work across servers).</p><p><strong>Webhook configuration</strong> — Setting up webhooks for delivery, bounce, open, click, spam complaint, and subscription change events. Your agent will understand the webhook payload structure and know how to handle each event type.</p><p><strong>Email best practices</strong> — Deliverability guidance, compliance requirements (CAN-SPAM, GDPR, CASL), transactional email design patterns, list management, safe testing strategies, and sending reliability. This one is useful even if you're not writing code! Ask your agent about SPF, DKIM, and DMARC setup, or what your legal obligations are for a marketing email list.</p><h2><strong>Why this matters</strong></h2><p>We've been building out AI tooling for Postmark over the past year. It started with our <a href="https://postmarkapp.com/lp/mcp">MCP server</a> for letting AI tools interact with your Postmark account directly. Then came our <a href="https://postmarkapp.com/llms.txt">llms.txt file</a> for giving AI assistants accurate API context. And we added <a href="https://postmarkapp.com/developer/integration/ai-prompts">pre-built AI prompts</a> to our developer documentation for common integration tasks.</p><p>Skills fill a different gap. Where the MCP server lets AI tools <i>do</i> things with Postmark (send emails, check stats), skills teach AI coding agents how to <i>write code</i> that uses Postmarks APIs and SDKs correctly. They're complementary! Together, they cover most of the ways developers use AI tools in their workflow today.</p><p>Here's a concrete example. Without skills, you might ask your agent to "set up Postmark email sending" and get code that works for a single email but doesn't handle batch sends, ignores Message Streams, and has no error handling. With the Postmark send-email skill loaded, your agent knows about batch chunking at 500-email limits, the difference between transactional and broadcast streams, which HTTP error codes to retry and which to fix, and how to handle partial failures in batch responses.</p><p>The difference is production-ready code versus a starting point you'll need to debug.</p><h2><strong>Getting started</strong></h2><p>Install all skills:</p>

      
    
      
        <pre><code class="language-bash">npx skills add ActiveCampaign/postmark-skills</code></pre>

      
    
      
        <p>Or install a specific skill:</p>

      
    
      
        <pre><code class="language-bash">npx skills add ActiveCampaign/postmark-skills --skill postmark-send-email</code></pre>

      
    
      
        <p>Once installed, your agent automatically uses the skills when relevant. You don't need to reference them explicitly, just ask your agent to "send a welcome email using Postmark" or "set up inbound email processing for a support ticket system" and it'll pull in the right context. Your agent will detect your project language and use the appropriate SDK.</p><h2><strong>What you can build</strong></h2><p>Once your agent has these skills, here are some things you can ask it to do:</p><p>"Send a welcome email using Postmark" — Your agent will use the correct SDK for your project, set up proper authentication, choose the right Message Stream, and include error handling.</p><p>"Set up inbound email processing for a support ticket system" — Your agent will configure the webhook endpoint, parse the MailboxHash for ticket routing, use StrippedTextReply for clean reply text, and handle attachments.</p><p>"Create a Postmark template for order confirmation emails" — Your agent will set up a Handlebars template with proper variable syntax, layout inheritance, and validation before deployment.</p><p>"Add bounce and delivery webhooks to track email status" — Your agent will configure the right webhook events, build a handler that processes each event type, and implement proper error handling.</p><h2><strong>Open source and ready for feedback</strong></h2><p>Postmark Skills is MIT licensed and <a href="https://github.com/ActiveCampaign/postmark-skills">available on GitHub</a>. If you run into an issue, find a missing edge case, or want to contribute an improvement, open a PR. This is the kind of thing that gets better with input from the people who actually use it.</p><p>If you're already using our MCP server or llms.txt file, skills are a natural next step. And if you're new to Postmark's AI tooling, this is a great place to start! Install the skills, point your AI agent at your next email integration task, and see what happens.</p><p>We'd love to hear what you build. Share your experiments with us on <a href="https://x.com/postmarkapp">X @postmarkapp</a> or drop us a line through <a href="https://postmarkapp.com/contact">support</a>.</p>
